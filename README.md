# hospital-management-system-live
The Hospital Management System (HMS) aims to streamline hospital operations by automating patient management, appointment scheduling, and communication among doctors, receptionists, and administrators. This system will leverage real-time data transfer technology to ensure instant updates between the receptionist and doctor interfaces.
#.	Objectives
•	Build a multi-user hospital management web application for admin, doctor, and receptionist roles.
•	Enable real-time data exchange (live updates) for appointments, patient check-ins, diagnoses, and records using technologies such as Socket.IO.
•	Provide secure authentication and role-based access control for all user types.
•	Streamline workflow through an intuitive and responsive interface.
•	Implement admin controls for user management, reporting, and system settings.
#	Scope
•	Covered: Patient record management, appointment/queue management, EHR access and updates, user authentication, real-time notifications, admin dashboard.
•	Excluded: Insurance, external billing integration, extensive inventory management.
•	User Groups: Hospital administrators (admin), doctors, receptionists.
•	Deployment: Suitable for cloud or local web server environments.
